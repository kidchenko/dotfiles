## 2024-05-24 - [Insecure Predictable Temporary File Path in Setup Script]
**Vulnerability:** The script `tools/os_installers/apt.sh` used a predictable, hardcoded temporary file path (`/tmp/yq`) for downloading the `yq` binary before moving it to `/usr/local/bin/yq`. This was performed using `wget -O /tmp/yq` followed by `sudo mv /tmp/yq /usr/local/bin/yq`.
**Learning:** Hardcoding predictable temporary file locations, especially in world-writable directories like `/tmp`, creates a high risk of symlink attacks or local privilege escalation. An attacker could pre-create `/tmp/yq` as a symlink pointing to a critical system file (e.g., `/etc/passwd`). When the script executes `sudo mv /tmp/yq ...`, it would overwrite the critical system file as root.
**Prevention:** Always use secure, dynamically generated temporary file paths (e.g., via `mktemp`) when downloading artifacts or creating intermediate files in shared directories. For elevated privileges, this is absolutely essential.
